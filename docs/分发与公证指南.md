# BOG Tool — 分发给其他 Mac 且不被拦截指南

当前现象：打包后的 app 分享给另一台 Mac 会被系统拦截（“无法打开，因为无法验证开发者”等）。  
**根本原因**：Apple 要求「Developer ID 签名 + 公证（Notarization）」后，其他电脑上的 Gatekeeper 才会放行。仅改分发格式（.app / .dmg / .zip）不能解决拦截问题。

下面从**账号未完善、从未成功发布**的情况，按顺序做完即可。

---

## 一、先完善 Apple Developer 账号（必做）

未填账单地址、未同意协议时，无法使用「Developer ID」签名和公证。

1. **登录**  
   - 打开 [Apple Developer](https://developer.apple.com/account) 用你的 Apple ID 登录。

2. **同意协议**  
   - 若首页有 **Agreements, Tax, and Banking** 或「需要更新协议」的提示，点进去。  
   - 在 **Agreements** 里把 **Paid Applications**（和/或当前要求同意的协议）**同意**并提交。

3. **填写账单与税务（付费开发者才需）**  
   - 若团队是 **Paid/付费** 开发者计划：  
     - 在 **Agreements, Tax, and Banking** 中完成 **Tax Forms**、**Banking**（收款）、**Contact** 等。  
   - 若团队是 **Free**：通常只需同意协议，没有账单地址也能用「Apple Development」在本机跑，但**分发给别人**必须用 **Developer ID**，而 Developer ID 证书一般需要**付费开发者计划**（$99/年）。  
   - 若你当前是 Free 且没有付费：只能在本机或通过 TestFlight 等受限方式分发；要直接发 .app 给别的 Mac 且不被拦，需要加入付费计划并完成上述信息。

4. **确认团队与证书**  
   - **Membership** 里确认状态是 **Active**。  
   - **Certificates, Identifiers & Profiles** → **Certificates** 里应有 **Developer ID Application**（用于给 Mac app 签名）。若没有，在 Xcode 里用该 Apple ID 登录后，用「Distribute App → Developer ID」导出时会提示创建。

完成以上后，才能稳定完成「Developer ID 签名 + 公证」。

---

## 二、在 Xcode 里用 Developer ID 打包导出

1. 用 **Release**、**Any Mac (Apple Silicon, Intel)**（或你需要的架构）在 Xcode 里 **Product → Archive**。  
2. Archive 完成后在 Organizer 里选中该 Archive，点 **Distribute App**。  
3. 选择 **Developer ID**（分发给其他 Mac，不走 App Store），下一步。  
4. 选项里勾选：  
   - **Upload your app’s symbols**（可选，便于崩溃分析）  
   - **Manage distribution certificate and profile** 选 **Automatically**（或按你们团队习惯）。  
5. 下一步直到选择 **Export**，导出到一个文件夹，得到 **BOG Tool.app**。

此时得到的是「已用 Developer ID 签名」的 .app，但**尚未公证**，发到别的 Mac 仍可能被拦。

---

## 三、公证（Notarization）— 解决“被拦截”的关键

公证是 Apple 对二进制做一次扫描并签发「可信任」票据；其他 Mac 只有在检测到有效公证后才会放行。

### 方式 A：Xcode 导出时勾选上传公证（推荐）

在 **Distribute App → Developer ID** 的某一步（取决于 Xcode 版本）会有选项：

- **Upload to notarization service** / **Upload for notarization**

勾选后，Xcode 会在导出后自动把 .app 上传给 Apple 公证；若失败会给出原因（如签名问题、依赖问题）。  
若成功，导出的 .app 已带公证，可直接把该 .app（或把它打成 zip/dmg）发给别人。

### 方式 B：命令行公证（导出后手动做）

若 Xcode 里没有勾选或想自己控制流程：

1. **安装/配置命令行工具**（一般 Xcode 已带）  
   ```bash
   xcrun notarytool --version
   ```

2. **先配置一次 Apple ID（若未配置）**  
   ```bash
   xcrun notarytool store-credentials "AC_PASSWORD" --apple-id "你的AppleID@email.com" --team-id "38V6W364PW" --password "app专用密码"
   ```  
   - `app专用密码`：在 [appleid.apple.com](https://appleid.apple.com) → 登录与安全 → App 专用密码 里生成。  
   - 之后公证时会用到配置名 `AC_PASSWORD`。

3. **提交公证**（对导出的 .app）  
   ```bash
   xcrun notarytool submit "/path/to/BOG Tool.app" --keychain-profile "AC_PASSWORD" --wait
   ```  
   - `--wait` 会等到公证完成；成功会打印 `status: Accepted`。

4. **把公证票据“钉”到 .app 上（推荐）**  
   这样别的 Mac 离线也能通过校验：  
   ```bash
   xcrun stapler staple "/path/to/BOG Tool.app"
   ```

5. **再分发**  
   把这份已 staple 的 **BOG Tool.app** 压缩成 zip 或放进 dmg，发给对方。对方解压/安装后，Gatekeeper 会放行。

若公证失败，`notarytool submit` 会返回 `status: Invalid` 并给出日志；根据日志修（常见：签名不完整、有未签名的嵌套内容、Hardened Runtime 未开等），再重新 Archive → 导出 → 公证。

---

## 四、对方电脑上若仍提示“已损坏”或无法打开

有时是本地缓存或旧版本残留：

1. 让对方在 **系统设置 → 隐私与安全性** 里看是否有「仍要打开」或「允许」该应用的按钮，点一次。  
2. 或在终端执行（将 `BOG Tool.app` 换成实际路径）：  
   ```bash
   xattr -cr "/Applications/BOG Tool.app"
   ```  
   再双击打开。

若你这边已按上面完成「Developer ID 签名 + 公证 + staple」，对方一般不会再有拦截。

---

## 五、总结对照

| 你当前情况                     | 结果说明 |
|--------------------------------|----------|
| 账号未完善（协议/账单未填）     | 无法完成公证或无法用 Developer ID，分发必被拦。 |
| 只改了分发格式（如改成 dmg）    | 不会解决拦截，Gatekeeper 看的是签名+公证。 |
| 完成账号 → Developer ID 导出 → 公证 → staple | 再分发 .app 或 .dmg，其他 Mac 可正常打开。 |

**建议顺序**：先到 [developer.apple.com/account](https://developer.apple.com/account) 把协议、账单（若为付费）等信息填完 → 再在 Xcode 里 Archive → Developer ID 导出（勾选公证或导出后命令行公证）→ 对 .app 做 staple → 用 zip 或 dmg 分享给另一台 Mac。
