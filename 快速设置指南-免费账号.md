# 快速设置指南 - 使用免费 Apple ID 运行应用

## 🎯 目标
让你的应用可以在自己的 Mac 上直接打开运行，**不需要付费开发者账号**！

## ⚡ 5 分钟快速设置

### 步骤 1：登录 Apple ID（2 分钟）

1. 打开 **Xcode**
2. 菜单栏：**Xcode > Settings**（或按 `Command + ,`）
3. 点击 **Accounts** 标签
4. 点击左下角的 **+** 号
5. 选择 **Apple ID**
6. 输入你的 Apple ID 和密码
   - ✅ 任何 Apple ID 都可以（不需要付费账号）
   - ✅ 如果没有，去 appleid.apple.com 免费注册一个
7. 登录成功后，你会看到：
   - 你的名字
   - 下面显示 "Personal Team" 或 "个人团队"

### 步骤 2：在项目中设置团队（2 分钟）

**详细步骤（如果找不到，请看 `如何找到Signing设置-详细步骤.md`）：**

1. **找到项目设置**：
   - 在 Xcode 中，看**左侧导航栏**（如果看不到，按 `Command + 1`）
   - 找到**最顶部**的**蓝色图标**，名字是 **BOG_TOOL**
   - **单击**这个蓝色图标（不是双击）

2. **选择 Target**：
   - 点击项目文件后，**中间区域**会显示设置
   - 找到 **TARGETS** 这个标题
   - 在 TARGETS 下面，找到 **BOG_TOOL**
   - **单击 BOG_TOOL**（不是项目文件，是 TARGETS 下面的）

3. **找到 Signing 设置**：
   - 选择 target 后，**顶部**会出现标签页
   - 点击 **Signing & Capabilities** 标签
   - 如果看不到这个标签，点击 **Build Settings**，然后搜索 "signing"

4. **设置团队**：
   - ✅ 勾选 **Automatically manage signing**
   - 在 **Team** 下拉菜单中选择你的名字（Personal Team）
   - 如果下拉菜单是空的，说明还没登录 Apple ID，回到步骤 1

5. **如果看到警告或错误**：
   - 点击 **Try Again** 按钮
   - 或者点击 **Manage Certificates**，然后点击 **+** 添加证书

### 步骤 3：重新构建应用（1 分钟）

1. 菜单栏：**Product > Clean Build Folder**（或按 `Shift + Command + K`）
2. 等待清理完成
3. 菜单栏：**Product > Build**（或按 `Command + B`）
4. 等待构建完成（应该没有错误）

### 步骤 4：测试运行

1. 在 Xcode 中运行：**Product > Run**（或按 `Command + R`）
   - ✅ 应该可以正常运行

2. 找到构建好的应用：
   - 在 Xcode 中，点击左侧导航栏的 **Products** 文件夹
   - 右键点击 **BOG Tool.app**
   - 选择 **Show in Finder**
   - 在 Finder 中双击应用
   - ✅ 应该可以直接打开了！

## ✅ 成功标志

如果设置成功，你应该能够：
- ✅ 在 Xcode 中正常运行应用
- ✅ 在 Finder 中直接双击 `.app` 文件打开应用
- ✅ 应用正常启动，没有 "已损坏" 或 "无法验证开发者" 的错误

## ❌ 常见问题

### 问题 1：找不到 Team 选项
**解决**：确保已登录 Apple ID（步骤 1）

### 问题 2：显示 "No signing certificate"
**解决**：
1. 点击 **Manage Certificates**
2. 点击左下角的 **+** 号
3. 选择 **Apple Development**
4. 关闭窗口，回到项目设置
5. 重新选择 Team

### 问题 3：应用还是打不开
**解决**：
1. 移除隔离属性：
   ```bash
   xattr -dr com.apple.quarantine "/path/to/BOG Tool.app"
   ```
2. 在系统设置中允许：
   - 系统设置 > 隐私与安全性
   - 如果看到应用被阻止，点击 "仍要打开"

### 问题 4：7 天后应用打不开了
**原因**：免费账号的签名有效期是 7 天
**解决**：在 Xcode 中重新构建一次即可（Product > Build）

## 📝 重要提示

### ✅ 免费账号可以做什么：
- 在自己的 Mac 上运行应用
- 使用所有功能（蓝牙、文件访问等）
- 开发和调试应用

### ⚠️ 免费账号的限制：
- 签名有效期 7 天（过期后重新构建即可）
- 只能在自己的 Mac 上运行
- 不能分发给其他人
- 不能发布到 App Store

### 💡 如果需要分发给别人：
- 需要付费的开发者账号（$99/年）
- 或者使用 Developer ID 证书进行签名和公证
